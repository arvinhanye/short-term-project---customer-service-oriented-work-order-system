# 工单管理系统用户手册

## 1. 适用对象

本文档适用于工单管理系统的普通用户、管理员和课程验收人员，说明系统安装、初始化、运行、登录、普通用户操作、管理员操作和常见问题处理方式。

## 2. 环境准备

运行前请准备：

| 项目 | 要求 |
| --- | --- |
| JDK | 17 或以上 |
| Maven | 3.8 或以上 |
| MySQL | 8.0 或以上 |
| MongoDB | 5.0 或以上 |

进入项目目录：

```bash
cd ticket-management
```

## 3. 数据库初始化

### 3.1 MySQL 初始化

先创建并初始化 MySQL 数据库，按以下顺序执行脚本：

```bash
mysql -u root -p < src/main/resources/sql/mysql_schema.sql
mysql -u root -p ticket_management < src/main/resources/sql/mysql_views.sql
mysql -u root -p ticket_management < src/main/resources/sql/mysql_procedures.sql
mysql -u root -p ticket_management < src/main/resources/sql/mysql_triggers.sql
mysql -u root -p ticket_management < src/main/resources/sql/mysql_init_data.sql
mysql -u root -p ticket_management < src/main/resources/sql/mysql_day07_optimization.sql
```

如果是首次初始化，前五个脚本为必要脚本；`mysql_day07_optimization.sql` 用于补充性能优化索引。

### 3.2 MongoDB 初始化

执行 MongoDB 初始化脚本：

```bash
mongosh < src/main/resources/sql/mongodb_init.js
```

脚本会创建工单详情、评论、行为日志和系统日志相关集合与索引。

### 3.3 修改连接配置

打开 `src/main/resources/db.properties`，根据本机环境修改：

```properties
mysql.url=jdbc:mysql://localhost:3306/ticket_management?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8
mysql.username=root
mysql.password=your_password
mongodb.uri=mongodb://localhost:27017
mongodb.database=ticket_management_logs
```

## 4. 构建与启动

常规测试：

```bash
mvn test
```

打包：

```bash
mvn clean package
```

启动桌面客户端：

```bash
java -jar target/ticket-management.jar
```

## 5. 默认账号

初始化脚本提供以下示例账号，默认密码均为：

```text
Ticket@123
```

| 角色 | 用户名示例 | 说明 |
| --- | --- | --- |
| 管理员 | `admin01`、`admin02`、`admin03`、`admin04`、`admin05` | 可进入管理员工作台 |
| 普通用户 | `user01`、`user02`、`user03`、`user04`、`user05`、`user07` | 可进入普通用户工作台 |
| 禁用用户 | `user06` | 用于验证禁用账号无法登录 |

## 6. 登录与注册

### 6.1 登录

1. 启动程序后进入登录页面。
2. 输入用户名和密码。
3. 点击“登录”。
4. 管理员账号进入管理员工作台，普通用户账号进入普通用户工作台。

登录页支持按 Enter 提交。注册窗口和主要业务弹窗可按 Esc 关闭。

### 6.2 注册普通用户

1. 在登录页点击“注册”。
2. 输入用户名、密码、邮箱和手机号。
3. 点击“提交注册”。
4. 注册成功后返回登录页。

密码要求：

| 要求 | 说明 |
| --- | --- |
| 长度 | 8 到 64 位 |
| 组成 | 必须包含小写字母、大写字母、数字和特殊字符 |
| 示例 | `Ticket@123` |

## 7. 普通用户操作

普通用户登录后进入“工单中心”，可看到“我的工单”、“创建工单”、“联系信息”三个页签。页头保留当前用户、刷新和退出入口。

### 7.1 查看我的工单

1. 打开“我的工单”页签。
2. 系统默认加载当前用户最近 20 条工单，列表展示工单编号、标题、分类、优先级、状态、金额、创建时间和更新时间。
3. 可通过状态下拉框和关键词输入框筛选工单，点击“搜索”执行查询，点击“重置”恢复全部状态。
4. 点击“上一页”“下一页”切换分页，点击顶部“刷新工单”可重新加载当前筛选结果。系统使用稳定游标翻页，筛选条件变化后会自动从第一页重新加载。
5. 双击工单行，或选中工单后按回车，可打开详情窗口。

详情窗口展示工单编号、标题、状态、金额、分类、优先级、创建时间、描述和沟通记录。沟通记录时间按北京时间展示。用户可在详情窗口追加回复；工单处理结束后，也可提交评分和评价内容。

列表、详情、沟通记录和管理员报表中的时间统一按 `年-月-日 时:分 北京时间` 显示，例如 `2026-07-09 15:12 北京时间`。

工单状态含义：

| 状态码 | 含义 |
| --- | --- |
| 0 | 待处理 |
| 1 | 处理中 |
| 2 | 已完成 |
| 3 | 已关闭 |
| 4 | 已取消 |

### 7.2 创建工单

1. 打开“创建工单”页签。
2. 系统在后台加载当前可用分类；在左侧紧凑表单填写标题、分类、金额、优先级和问题描述，右侧会显示填写提示。
3. 点击“提交工单”。
4. 按钮显示“提交中”期间请勿重复操作；提示“工单创建成功”后，可回到“我的工单”查看新工单。

标题、分类和问题描述为必填项。缺少内容时，对应输入框会显示红色边框，窗口右下角会列出尚未填写的项目；补充内容后红框自动消失，不会弹出阻断操作的对话框。

系统输入快捷键统一如下：单行输入框按 Enter 执行当前页面的查询、登录、保存或提交；问题描述、联系备注、客户回复、评价、客服回复和内部备注等多行输入框按 Enter 提交，按 Shift+Enter 换行。

常用分类：

| 分类 |
| --- | --- |
| 账户问题 |
| 支付问题 |
| 系统故障 |
| 订单咨询 |
| 登录异常 |
| 实名认证 |
| 退款申请 |
| 重复扣费 |
| 页面报错 |
| 功能异常 |

优先级可选择：

| 优先级 | 含义 |
| --- | --- |
| `LOW` | 低 |
| `MEDIUM` | 中 |
| `HIGH` | 高 |
| `URGENT` | 紧急 |

### 7.3 查看推荐分类

1. 打开“创建工单”页签。
2. 点击“推荐分类”。
3. 系统会根据当前用户历史工单返回最近常用分类推荐。

### 7.4 维护联系信息

1. 打开“联系信息”页签。
2. 修改姓名/称呼、邮箱、手机号、首选联系方式、通知偏好、服务地址和备注。
3. 点击“保存资料”。
4. 如需实名，点击“实名认证”按钮，在弹窗中填写真实姓名和证件号码。
5. 实名认证完成后，按钮显示为“已实名认证”并置灰。
6. 系统提示“资料已保存”后生效。

管理员可在“全部工单”中打开工单详情，查看该工单提交人的客户资料。当前展示内容包括姓名/称呼、手机号和服务地址。

### 7.5 退出登录

点击顶部“退出登录”，系统会关闭管理员模块窗口、清空当前账号的列表和表单缓存，并返回已清空凭据的登录页面。

## 8. 管理员操作

管理员登录后进入管理员工作台。顶部提供“查看统计”、“用户管理”、“更多操作”和“退出登录”；“更多操作”中包含系统自检、连接池监控和批量取消超时。左侧模块导航单击即可进入工单、分类、日志和连接池功能。

“全部工单”窗口每页显示最多 50 条，可使用“上一页”“下一页”浏览全部结果。标题关键词与状态条件由数据库分页查询处理；切换条件时自动返回第一页。

### 8.1 查看统计

1. 点击“查看统计”。
2. 系统打开“数据统计与报表”窗口。报表和聚合数据在后台加载，状态栏会显示“加载中”。
3. 在“月度报表”页签选择年份、月份，点击“刷新”。
4. 在“MongoDB 聚合统计”页签查看行为类型、近 30 天趋势、热门工单、用户活跃度、客户端分布、评分分布、评论标签、工单评论和最近行为。行为、评论与审计类汇总默认聚合近 30 天数据，排行榜最多显示 20 条。
5. 在“系统日志审计”页签按类型、级别、用户 ID、关键词和条数筛选日志，也可查看类型汇总、级别汇总、用户汇总和近 30 天趋势。

### 8.2 用户管理

1. 点击“用户管理”。用户列表会在后台加载。
2. 可按用户名/邮箱和账号状态筛选用户。
3. 选中用户后，可点击“启用账号”或“禁用账号”；系统会进行管理员权限校验并记录审计日志。
4. 为避免当前会话失效，管理员不能在当前窗口中禁用自己的账号。

### 8.3 系统自检

1. 点击“系统自检”。检查在后台执行，避免界面在数据库响应期间失去响应。
2. 系统依次检查 MySQL 连接、MongoDB 连接、分类 DAO 查询和工单分页查询。
3. 右侧区域显示“系统状态：稳定”或异常提示。

### 8.4 连接池监控

1. 点击“连接池监控”。
2. 系统打开“连接池状态监控”窗口。
3. 查看 HikariCP 连接池名称、最大连接数、正在使用连接、空闲连接、总连接数、等待连接线程和使用率。
4. 点击“模拟占用连接”，系统会临时占用 3 条 MySQL 连接 8 秒，窗口自动刷新时可看到“正在使用连接”和“使用率”上升。
5. 如果“等待连接线程”大于 0 或使用率长期接近 100%，说明连接池可能出现拥塞，应检查慢查询、事务未释放或连接池容量配置。

### 8.5 批量取消超时工单

1. 点击“批量取消超时”。
2. 确认提示：“将把 30 天前仍为待处理的工单批量流转为已取消，是否继续？”
3. 点击“是”后，系统调用存储过程批量处理。
4. 右侧区域显示影响的工单数量。

### 8.6 退出登录

点击顶部“退出登录”，系统返回登录页面。

## 9. 验收建议

建议按以下顺序验收：

1. 按脚本初始化 MySQL 和 MongoDB。
2. 执行 `mvn test`，确认单元测试通过。
3. 执行 `mvn clean package`，确认打包成功。
4. 使用 `admin01 / Ticket@123` 登录管理员工作台，验证统计、用户管理、系统自检和批量维护入口。
5. 使用 `user01 / Ticket@123` 登录普通用户工作台，验证我的工单、创建工单、推荐分类和个人资料保存。
6. 使用 `user06 / Ticket@123` 登录，验证禁用用户无法进入系统。

## 10. 常见问题

| 问题 | 处理方式 |
| --- | --- |
| 登录提示失败 | 检查用户名、密码、账号状态和 MySQL 初始化数据 |
| 启动后数据库连接失败 | 检查 `db.properties`、MySQL/MongoDB 服务状态和端口 |
| 创建工单提示分类不存在 | 使用初始化脚本中的分类 ID，或确认 `categories` 表存在数据 |
| 月度报表为空 | 确认所选年月有工单数据，初始化数据主要集中在 2026 年 2 月和 3 月 |
| MongoDB 聚合为空 | 确认已执行 `mongodb_init.js`，并在系统中执行过登录、查看、创建等操作 |
| 压力测试未执行 | `ActionLogStressTest` 默认跳过，需要添加 `-Dstress=true` |

## 11. 压力测试

压力测试会向 MongoDB 写入测试行为日志，请确认连接到测试库后执行：

```bash
mvn -Dstress=true -Dtest=com.ticket.performance.ActionLogStressTest test
```

成功时会输出写入总量、并发数和耗时。
