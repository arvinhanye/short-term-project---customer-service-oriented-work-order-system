# 工单管理系统用户手册

## 1. 适用对象

本文档适用于工单管理系统的普通用户、管理员和课程验收人员，说明系统安装、初始化、运行、登录、普通用户操作、管理员操作和常见问题处理方式。

## 2. 环境准备

运行前请准备：

| 项目 | 要求 |
| --- | --- |
| JDK | 17 或以上 |
| Maven | 3.8 或以上 |
| MySQL | 8.0 或以上 |
| MongoDB | 5.0 或以上 |

进入项目目录：

```bash
cd ticket-management
```

## 3. 数据库初始化

### 3.1 MySQL 初始化

先创建并初始化 MySQL 数据库，按以下顺序执行脚本：

```bash
mysql -u root -p < src/main/resources/sql/mysql_schema.sql
mysql -u root -p ticket_management < src/main/resources/sql/mysql_views.sql
mysql -u root -p ticket_management < src/main/resources/sql/mysql_procedures.sql
mysql -u root -p ticket_management < src/main/resources/sql/mysql_triggers.sql
mysql -u root -p ticket_management < src/main/resources/sql/mysql_init_data.sql
mysql -u root -p ticket_management < src/main/resources/sql/mysql_day07_optimization.sql
```

如果是首次初始化，前五个脚本为必要脚本；`mysql_day07_optimization.sql` 用于补充性能优化索引。

旧版本数据库升级时，备份后再额外执行一次：

```bash
mysql -u root -p ticket_management < src/main/resources/sql/mysql_password_security.sql
```

### 3.2 MongoDB 初始化

执行 MongoDB 初始化脚本：

```bash
mongosh < src/main/resources/sql/mongodb_init.js
```

脚本会创建工单详情、评论、行为日志和系统日志相关集合与索引。

### 3.3 修改连接配置

先由数据库管理员创建专用应用账号，不要让桌面客户端使用 `root`：

```sql
CREATE USER 'ticket_app'@'localhost' IDENTIFIED BY '请替换为独立强密码';
GRANT SELECT, INSERT, UPDATE, DELETE, EXECUTE ON ticket_management.* TO 'ticket_app'@'localhost';
```

运行前设置环境变量；密码不会写入仓库或打包进 JAR：

```bash
export TICKET_MYSQL_USERNAME=ticket_app
export TICKET_MYSQL_PASSWORD='上一步设置的独立强密码'
```

macOS 双击启动器时默认使用 `ticket_app`，优先从登录钥匙串的 `ticket-management-mysql` 项读取密码；未找到时只询问该账号的密码，并会在启动 Java 前验证。Windows 启动器要求提前设置变量。读写库分开部署时，可使用 `TICKET_MYSQL_WRITE_*` 和 `TICKET_MYSQL_READ_*` 分别配置。应用拒绝空密码和 `root` 账号。

## 4. 构建与启动

常规测试：

```bash
mvn test
```

```bash

```

启动桌面客户端：

```bash

```

| 角色 | 验收账号 | 初始临时密码 | 说明 |
| --- | --- | --- | --- |
| 管理员 | `admin01` | `CedarFalcon#481` | 首次登录强制换密 |

## 6. 登录与注册

### 6.1 登录

1. 启动程序后进入登录页面。
2. 输入用户名和密码。
3. 点击“登录”。
4. 管理员账号进入管理员工作台，普通用户账号进入普通用户工作台。

登录页支持按 Enter 提交。注册在当前页面内完成，可按 Esc 返回登录；主要业务弹窗仍可按 Esc 关闭。

### 6.2 注册普通用户

1. 在登录页点击“创建账号”，页面会原地切换为注册表单。
2. 输入用户名、密码、确认密码、邮箱和手机号。
3. 点击“提交注册”。
4. 注册成功后页面自动切回登录表单，并带入新用户名。

密码要求：

| 要求 | 说明 |
| --- | --- |
| 长度 | 12 到 64 位，且 BCrypt UTF-8 编码不超过 72 字节 |
| 组成 | 必须包含小写字母、大写字母、数字和特殊字符 |
| 限制 | 不能使用常见密码、系统名称、用户名、邮箱前缀或手机号 |

## 7. 普通用户操作

普通用户登录后进入“工单中心”，可看到“我的工单”、“创建工单”、“联系信息”三个页签。页头保留当前用户、刷新、修改密码和退出入口。修改密码时必须验证当前密码，新密码不得与当前密码相同。

### 7.1 查看我的工单

1. 打开“我的工单”页签。
2. 系统默认加载当前用户最近 20 条工单，列表展示工单编号、标题、分类、优先级、状态、金额、创建时间和更新时间。
3. 可通过状态下拉框和关键词输入框筛选工单，点击“搜索”执行查询，点击“重置”恢复全部状态。
4. 点击“上一页”“下一页”切换分页，点击顶部“刷新工单”可重新加载当前筛选结果。系统使用稳定游标翻页，筛选条件变化后会自动从第一页重新加载。
5. 双击工单行，或选中工单后按回车，可打开详情窗口。

详情窗口展示工单编号、标题、状态、金额、分类、优先级、创建时间、描述和沟通记录。沟通记录时间按北京时间展示。用户可在详情窗口追加回复；工单处理结束后，也可提交评分和评价内容。

列表、详情、沟通记录和管理员报表中的时间统一按 `年-月-日 时:分 北京时间` 显示，例如 `2026-07-09 15:12 北京时间`。

工单状态含义：

| 状态码 | 含义 |
| --- | --- |
| 0 | 待处理 |
| 1 | 处理中 |
| 2 | 已完成 |
| 3 | 已关闭 |
| 4 | 已取消 |

### 7.2 创建工单

1. 打开“创建工单”页签。
2. 系统在后台加载当前可用分类；在左侧紧凑表单填写标题、分类、金额、优先级和问题描述，右侧会显示填写提示。
3. 点击“提交工单”。
4. 按钮显示“提交中”期间请勿重复操作；提示“工单创建成功”后，可回到“我的工单”查看新工单。

标题、分类和问题描述为必填项。缺少内容时，对应输入框会显示红色边框，窗口右下角会列出尚未填写的项目；补充内容后红框自动消失，不会弹出阻断操作的对话框。

系统输入快捷键统一如下：单行输入框按 Enter 执行当前页面的查询、登录、保存或提交；问题描述、联系备注、客户回复、评价、客服回复和内部备注等多行输入框按 Enter 提交，按 Shift+Enter 换行。

常用分类：

| 分类 |
| --- | --- |
| 账户问题 |
| 支付问题 |
| 系统故障 |
| 订单咨询 |
| 登录异常 |
| 实名认证 |
| 退款申请 |
| 重复扣费 |
| 页面报错 |
| 功能异常 |

优先级可选择：

| 优先级 | 含义 |
| --- | --- |
| `LOW` | 低 |
| `MEDIUM` | 中 |
| `HIGH` | 高 |
| `URGENT` | 紧急 |

### 7.3 查看推荐分类

1. 打开“创建工单”页签。
2. 点击“推荐分类”。
3. 系统会根据当前用户历史工单返回最近常用分类推荐。

### 7.4 维护联系信息

1. 打开“联系信息”页签。
2. 修改姓名/称呼、邮箱、手机号、首选联系方式、通知偏好、服务地址和备注。
3. 点击“保存资料”。
4. 如需实名，点击“实名认证”按钮，在弹窗中填写真实姓名和证件号码。
5. 实名认证完成后，按钮显示为“已实名认证”并置灰。
6. 系统提示“资料已保存”后生效。

管理员可在“全部工单”中打开工单详情，查看该工单提交人的客户资料。当前展示内容包括姓名/称呼、手机号和服务地址。

### 7.5 退出登录

点击顶部“退出登录”，系统会关闭管理员模块窗口、清空当前账号的列表和表单缓存，并返回已清空凭据的登录页面。

## 8. 管理员操作

管理员登录后进入管理员中心。顶部显示当前管理员并提供“修改密码”和“退出登录”；左侧导航按“业务管理、数据分析、系统工具”分组，单击即可进入工单、用户、分类、报表、日志、自检、连接池和批量维护功能。工作概览只统计当前分配给登录管理员的工单：顶部显示当前待办、待处理、处理中和累计已完成；下方环形图展示个人工单状态，悬停可查看数量和占比，点击状态可跳转到对应的“我的工单”筛选。待办风险表按优先级和等待时间排序，双击可查看详情；快捷操作可进入个人待处理、处理中或全部个人工单，并可刷新概览。

“全部工单”窗口顶部集中提供标题、状态和分配客服筛选以及刷新按钮；左侧为工单列表和分页，右侧为详情与处理操作。每页最多显示 50 条，可使用列表底部的“上一页”“下一页”浏览全部结果。标题、状态与分配客服条件会先完成跨库筛选再分页，页脚显示当前筛选的准确总数；切换条件时自动返回第一页。选中工单后可点击“查看详情”，也可双击工单行打开完整详情。

### 8.1 查看统计

1. 点击左侧“数据报表”。
2. 系统打开“数据统计与报表”窗口。报表和聚合数据在后台加载，状态栏会显示“加载中”。
3. 在“月度报表”页签选择年份、月份并点击“刷新”。上方指标卡显示工单总数、各状态数量、平均金额和统计周期，下方保留详细统计口径。
4. 在“行为日志统计”页签从顶部选择行为类型、近 30 天趋势、热门工单、用户活跃度、客户端分布、评分分布、评论标签、工单评论或最近行为；结果表使用中文表头并占据主要空间。行为、评论与审计类汇总默认聚合近 30 天数据，排行榜最多显示 20 条。
5. 在“系统日志审计”页签通过第一行设置类型、级别、用户 ID、关键词和条数，通过第二行执行查询、重置或类型/级别/用户/趋势快速汇总。
6. 查询无结果时表格显示“暂无数据”；月度、行为和系统日志的加载或失败状态分别显示，不会互相覆盖。

### 8.2 用户管理

1. 点击左侧“用户管理”。用户列表会在后台加载。
2. 顶部可按用户名/邮箱和账号状态筛选用户；点击“重置”可恢复全部账号，列表标题同时显示账号总数、启用数、禁用数和当前结果数。
3. 左侧列表用于浏览账号，选中后右侧显示用户 ID、角色、状态、密码状态、邮箱、手机号和创建时间。
4. 在右侧点击“启用账号”“禁用账号”或“重置为临时密码”；重置产生的临时密码仅显示一次，用户下次登录必须立即换密。
5. 当前登录账号会标注为“当前账号”，禁用按钮不可用；即使绕过界面调用，`UserService` 也会拒绝自我禁用。

### 8.3 分类管理

1. 点击左侧“分类管理”。
2. 分类固定为两级：一级分类用于业务归组，二级分类用于具体问题；不支持三级及更深层级。
3. 左侧只读列表显示分类编号、名称、层级和父分类；历史数据若超过两级会标记为“层级异常”，点击“刷新”可重新加载。
4. 点击“新建分类”清空右侧表单，填写名称；父分类不选择时创建一级分类，选择一级分类时创建二级分类。父分类下拉框不会显示二级分类。
5. 选中左侧分类后，可在右侧修改名称或父分类并点击“保存修改”，也可点击“删除分类”。仍有二级分类的一级分类不能降级为二级分类。
6. 系统会拒绝创建三级分类，并禁止删除仍有二级分类或关联工单的分类。
7. 删除确认框会显示在分类管理窗口之上；确认或取消后仍返回分类管理窗口，不会将管理员主页面同时前置。

普通用户创建工单、查看工单详情或查看推荐分类时，二级分类统一显示为“一级分类 › 二级分类”，一级分类显示为“一级分类｜名称”。层级异常的历史分类不会出现在新工单选择项和推荐结果中，但旧工单详情仍保留异常路径，便于追溯。

### 8.4 系统自检

1. 点击左侧“系统自检”，系统会打开独立自检窗口并在后台执行检查。
2. 顶部状态卡显示系统状态、通过项目、失败项目和总耗时。
3. 明细表依次显示 MySQL 写库连接、MySQL 读库连接、MongoDB 连接、分类 DAO 查询和工单分页查询的状态、耗时及说明。
4. 点击“重新检查”可再次执行；重复执行会取消旧任务，关闭窗口后不会继续回写界面。
5. 全部通过时主页面显示“系统状态：稳定”；存在失败时可根据明细表中的失败原因检查数据库服务或配置。

### 8.5 连接池监控

1. 点击左侧“连接池监控”。
2. 系统打开“连接池状态监控”窗口。
3. 顶部 WRITE/READ 状态卡显示连接池状态、使用率、正在使用连接、最大连接、空闲连接和等待线程，每 1 秒自动刷新。
4. 在“实时指标”页签逐列比较读写池状态，在“参数配置”页签查看池名称、最小空闲连接和各类超时配置。
5. 点击“模拟占用连接”，系统会尝试临时占用最多 3 条写连接 8 秒；实际数量会根据当前负载缩减，并至少保留一条业务余量。自动刷新期间可以观察使用率变化。
6. “负载较高”表示使用率达到 80%；“存在等待”或“连接已满”表示可能拥塞，应检查慢查询、事务未释放或连接池容量配置。
7. 关闭窗口会停止自动刷新并取消仍在进行的模拟占用，连接会在任务结束时归还。

### 8.6 批量取消超时工单

1. 点击左侧“批量维护”。
2. 确认提示：“将把 30 天前仍为待处理的工单批量流转为已取消，是否继续？”
3. 点击“是”后，系统调用存储过程批量处理。
4. 操作完成后，批量维护页面显示实际影响的工单数量。

### 8.7 退出登录

点击顶部“退出登录”，系统返回登录页面。

## 10. 常见问题

| 问题 | 处理方式 |
| --- | --- |
| 登录提示失败 | 检查用户名、密码、账号状态和 MySQL 初始化数据 |
| 启动后数据库连接失败 | 检查 `db.properties`、MySQL/MongoDB 服务状态和端口 |
| 创建工单提示分类不存在 | 使用初始化脚本中的分类 ID，或确认 `categories` 表存在数据 |
| 月度报表为空 | 确认所选年月有工单数据，初始化数据主要集中在 2026 年 2 月和 3 月 |
| MongoDB 聚合为空 | 确认已执行 `mongodb_init.js`，并在系统中执行过登录、查看、创建等操作 |
| 压力测试未执行 | `ActionLogStressTest` 默认跳过，需要添加 `-Dstress=true` |

## 11. 压力测试

压力测试会向 MongoDB 写入测试行为日志，请确认连接到测试库后执行：

```bash
mvn -Dstress=true -Dtest=com.ticket.performance.ActionLogStressTest test
```

成功时会输出写入总量、并发数和耗时。
