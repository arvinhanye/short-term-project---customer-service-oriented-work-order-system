# 维护更新记录：分页、跨库查询与工作台体验

## 1. 更新目标

本次维护聚焦于工单列表查询性能、MongoDB 聚合查询边界和 Swing 桌面端响应体验。在不改变既有 Java Swing、JDBC、MySQL、MongoDB、DAO 与 Service 分层架构的前提下，优化列表与报表的读取路径。

## 2. 查询与分页优化

- `OrderDAO` 增加工单摘要联表查询，一次读取订单、工单、分类和提交人信息。
- `DetailDAO` 增加按工单 ID 集合的批量详情查询，`CrossDatabaseQueryService` 再将当前页详情合并到摘要结果，避免逐行跨库查询。
- 普通用户工单列表新增游标分页 DTO，以 `created_at` 和 `order_id` 作为稳定游标；每页多取一条记录判断下一页是否存在。
- 标题关键词、状态和用户范围仍由参数化 SQL 条件处理，保持现有权限隔离与分页筛选能力。

## 3. MongoDB 聚合边界

- 行为日志、评论和系统日志的默认聚合窗口调整为最近 30 天。
- 热门工单、用户活跃度、行为类型、客户端分布、评分、评论标签与用户审计汇总默认最多返回 20 条。
- 保留既有 MongoDB Aggregation Pipeline、索引与数据模型，不改变课程要求的数据源分工。

## 4. Swing 体验修复

- 管理员工单、用户列表、系统自检和统计报表读取统一放入 `SwingWorker`，避免在事件分发线程中执行数据库访问。
- 退出登录时清理普通用户与管理员工作台缓存，关闭已打开的管理员模块窗口，并清空登录凭据。
- 普通用户切换筛选条件时会重置游标，避免不同条件之间复用旧分页位置。

## 6. 验证建议

```bash
mvn test
mvn clean package
```

初始化 MySQL 和 MongoDB 后，使用普通用户验证筛选、翻页和退出后重新登录；使用管理员验证工单列表、统计报表、用户列表和系统自检在数据加载期间界面仍可操作。
